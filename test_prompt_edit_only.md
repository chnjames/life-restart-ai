# prompt_edit_only 角色参数集成测试

## 修改说明

在 `src/services/openai.ts` 中对 `generateSmartEditPrompt` 方法进行了重大改进，现在能够根据角色的基础参数生成更准确的图像编辑指令。

## 主要改进

### 1. 新增角色参数支持
- **性别**: 男性/女性，影响编辑指令的性别特征描述
- **发型**: 如"黑色长发"、"短发"等，确保编辑时保持发型一致性  
- **脸型**: 如"鹅蛋脸"、"方脸"等，保持面部特征一致
- **肤色**: 如"自然"、"白皙"等，确保肤色编辑一致性
- **体型**: 如"中等"、"纤细"等，保持体型特征
- **镜头**: 如"50mm 半身"，确保镜头构图一致

### 2. 示例对比

#### 修改前的 prompt_edit_only
```
"仅调整光线为更温暖的色调，人物和构图保持不变"
```

#### 修改后的 prompt_edit_only
```
"仅调整光线为更温暖的色调，人物和构图保持不变（人物特征：女性、黑色长发、鹅蛋脸）"
```

### 3. 智能特征描述

系统会根据角色信息自动生成特征描述：
- 只包含非默认值的特征（避免冗余）
- 自动格式化为便于AI理解的描述
- 与编辑指令无缝集成

### 4. 混合模式增强

保持原有的"AI创造 + 规则引导"混合模式：
- **30% AI创造**: "根据当前情境，仅微调一个能增强故事感的视觉元素（考虑成年人阶段、校园场景）（人物特征：女性、黑色长发、鹅蛋脸）"
- **70% 规则库**: "仅将场景改为现代化校园环境，人物形象保持不变（人物特征：女性、黑色长发、鹅蛋脸）"

## 预期效果

1. **图像一致性提升**: 确保角色在不同事件中保持外观一致
2. **编辑精度提高**: AI能更准确地理解需要保持的角色特征
3. **用户体验优化**: 生成的图像更符合用户创建的角色设定
4. **减少编辑失误**: 避免因角色特征描述不明确导致的图像错误

## 技术实现

- 修改了 `generateSmartEditPrompt` 方法签名，增加 `characterInfo` 参数
- 新增 `generateCharacterDescription` 方法，智能生成角色特征描述
- 更新了所有调用点，从游戏输入中提取角色信息
- 保持向后兼容，当没有角色信息时使用默认值
